# Rentoza Documentation

This directory contains the architectural documentation, implementation plans, and migration guides for the Rentoza platform.

## Directory Structure

### 🏗️ [Architecture](./architecture/)
High-level design documents, security audits, and system-wide diagnostic reports.
- **Key Files:** `CHECK_IN_COMPLETE_ARCHITECTURE.md`, `SECURITY_AUDIT_REPORT.md`

### 🚀 [Features](./features/)
Detailed implementation plans and status reports for specific feature sets.

- **[Check-In](./features/check-in/)**: Documentation for the guest/host check-in and handshake workflows.
- **[Guest Access](./features/guest-access/)**: Plans regarding public endpoints and guest permissions.
- **[Owner Portal](./features/owner-portal/)**: Enhancements and features specific to the vehicle owner experience.
- **[Search & Availability](./features/search-and-availability/)**: Logic and plans for vehicle search and availability filtering.

### 🔄 [Migrations](./migrations/)
Guides and progress reports for database and system migrations.
- **Key Files:** `GEOSPATIAL_LOCATION_MIGRATION_PLAN.md`

*Note: Please ensure new documentation is placed in the appropriate subdirectory to maintain organization.*
